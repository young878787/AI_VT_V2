# AI_VT_V2 功能提案池與明日行動

更新日期：2026-04-14

## 1. 功能提案池（依價值與成本評估）

評分方式：
- 使用者價值：高 / 中 / 低
- 實作成本：高 / 中 / 低
- 建議階段：P0 / P1 / P2

| 功能 | 使用者價值 | 實作成本 | 建議階段 | 說明 |
|---|---|---|---|---|
| 啟動健康檢查面板 | 高 | 低 | P0 | 一眼看懂 API、WebSocket、模型、TTS 是否正常 |
| 會話恢復與切換 | 高 | 中 | P0 | 避免斷線或重整後體驗斷裂 |
| 記憶管理頁（檢視/刪除/修正） | 高 | 中 | P0 | 提升記憶可信度與可控性 |
| 對話追蹤 ID + 階段指示 | 高 | 中 | P0 | 排錯效率大幅提升 |
| 成本儀表（token/回合） | 高 | 中 | P0 | 避免模型成本失控 |
| 語音輸入 STT | 高 | 中 | P1 | 完整語音互動閉環 |
| Persona 模式切換 | 中高 | 低中 | P1 | 讓角色有「可切換個性」 |
| 快捷指令列 | 中高 | 低 | P1 | 快速重送、精簡回覆、風格切換 |
| 直播模式 UI | 中 | 中 | P1 | 減少控制面板干擾、提升展示效果 |
| 劇情模式（場景腳本） | 中 | 中高 | P2 | 從聊天進化到互動內容 |
| 多角色輪替 | 中 | 高 | P2 | 內容更豐富，但開發複雜度高 |
| 外部工具整合（行程/提醒） | 中 | 中高 | P2 | 增加日常黏性與實用性 |

## 2. 建議先做的三個「高 CP」功能

## A. 啟動健康檢查面板（P0）

原因：
- 最低成本，立刻減少「不知道哪裡壞」的焦慮。

成功指標：
- 啟動後 5 秒內能顯示 5 個模組狀態（Live2D、WS、LLM、TTS、記憶）。

## B. 會話恢復與切換（P0）

原因：
- 直接提升連續使用意願，是陪聊產品的底盤。

成功指標：
- 重整後保留最近對話，且可手動開新會話。

## C. 語音輸入 STT（P1）

原因：
- 你的系統已經有 TTS 輸出，補上 STT 就形成完整語音循環。

成功指標：
- 一鍵說話、即時轉文字、送出對話並收到語音回覆。

## 3. 明天可直接開工的任務包（不需要一次做大）

## 任務包 1（60-90 分鐘）

目標：完成「健康檢查資訊規格」

輸出：
- 列出要檢查的模組清單
- 每個模組的判定條件
- 前端顯示文案（成功/失敗/建議操作）

## 任務包 2（60-90 分鐘）

目標：完成「會話生命週期規格」

輸出：
- 何時建立 session
- 何時保存、何時清除
- 使用者操作（新會話、續聊、重置）

## 任務包 3（60-90 分鐘）

目標：完成「記憶品質規則 v1」

輸出：
- 什麼內容值得寫入長期記憶
- 哪些記錄應去重
- 怎麼標記重要程度

## 4. 你現在可以用的決策模板（防失焦）

新功能進來時，只問四個問題：

1. 這功能是否讓使用者更常打開 VTuber？
2. 這功能是否讓對話更連續、更有陪伴感？
3. 這功能是否降低你自己排錯/維運成本？
4. 不做它，MVP 還能成立嗎？

如果第 1、2、3 都偏弱，先放提案池，不進本週待辦。

---

## 5. 最後提醒

你現在的狀態不是「沒目標」，而是「目標太多，缺少產品邊界」。

先把 P0 補齊，你的系統就會從技術展示跨到可持續使用。
等 P0 穩住，再開 P1，你會更有成就感，也更容易持續做下去。
