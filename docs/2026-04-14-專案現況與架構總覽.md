# AI_VT_V2 專案現況與架構總覽

更新日期：2026-04-14

## 1. 目前產品定位（從程式碼反推）

目前系統已經是「可互動的 AI VTuber 原型」，核心能力完整，但還偏開發者導向：

- 已能即時聊天（WebSocket 串流）
- 已能讓 AI 驅動 Live2D 表情與頭部動態
- 已有 TTS 播放與口型同步
- 已有基本長期記憶（user_profile.json + memory.md）
- 已有上下文壓縮機制避免 token 爆量

一句話總結：
你已經把最難的引擎層打通了，下一步不是「能不能動」，而是「能不能穩定、好用、可持續使用」。

---

## 2. 高層架構

### 前端（vtuber-web-app）

- React + TypeScript + Zustand
- Live2D 核心整合在 src/live2d
- 主要 UI 元件：
  - Live2DCanvas：WebGL 初始化與渲染循環
  - AIChatPanel：聊天輸入與串流輸出
  - ControlPanel：模型切換、動作測試、麥克風/視線追蹤開關
  - ModelParamPanel：即時參數觀測與手動覆寫
  - HitAreaOverlay：點擊區域可視化調試

### 後端（backend）

- FastAPI + WebSocket（/ws/chat）
- OpenAI 相容客戶端，支援 provider 切換（OpenRouter / Nvidia）
- Tool call 三件組：
  - set_ai_behavior
  - update_user_profile
  - save_memory_note
- 記憶系統：user_profile.json、memory.md、可選 session 檔案
- Token 估算 + 自動壓縮舊對話
- TTS：Google Cloud Chirp 3 HD（可選）

---

## 3. 核心資料流（現在實際運作）

1. 使用者在前端輸入訊息
2. wsService 傳送到後端 WebSocket
3. 後端組裝動態 system prompt（含使用者畫像與共同回憶）
4. 呼叫 LLM，允許 tool calls
5. 後端先回傳 behavior（表情參數），再串流文字
6. 前端同步更新 Live2D 表情 + 聊天內容
7. stream_end 後，後端背景合成 TTS，前端播放語音並做口型同步
8. 累積到閾值時執行 context 壓縮

## 4. 目前已完成的「硬實力」

### A. Live2D 技術整合（完成度高）

- Renderer 初始化順序正確（createRenderer → startUp → loadShaders → setupTextures）
- 動作載入後有呼叫 motion.setEffectIds（避免 SDK null crash）
- AI 行為參數有平滑插值與退場機制
- 提供模型拖移、縮放、視線追蹤、hit area 調試

### B. 對話引擎（完成度中高）

- 串流輸出已接通
- 工具呼叫與二次生成（拿最終文字）流程已打通
- XML tool_call 也有兼容處理
- 可依 provider 切換，且有 fail-fast 檢查

### C. 記憶系統（完成度中）

- profile + memory log 雙軌持久化
- 可選 session 歷史持久化
- 有 context 壓縮防爆機制

### D. 音訊體驗（完成度中）

- 後端 TTS 可啟用
- 前端 TTS 播放 + analyser 口型同步
- 麥克風即時 lip sync 已可用

## 5. 現階段主要瓶頸（不是 bug，而是產品化缺口）

1. 產品流程仍偏「開發模式」
- 有很多高階控制，但缺少新使用者可直接上手的導引流程。

2. 缺少「可觀測性」
- 沒有對話級別的追蹤 ID、成本統計、錯誤分層報告面板。

3. 記憶品質仍偏原始
- 目前偏 append 型儲存，缺少去重、衝突解決、重要性評分。

4. 缺少語音輸入閉環
- 目前是「文字輸入 + TTS 輸出」，尚未形成「說話 -> AI 回話」完整語音交互。

5. 缺少最小營運能力
- 沒有權限隔離、多使用者策略、基本安全欄（速率限制、token budget、敏感詞策略）。

---

## 6. 你現在的階段判定

你不是「沒目標」，你是已經過了最難的底層整合期，進入了典型的第二階段：

- 第一階段：把技術打通（你已做到了）
- 第二階段：定義 MVP 邊界，讓功能變成可持續使用的產品（你現在正在這裡）

建議心法：
接下來每個功能都先問一句：「它是否讓使用者更願意每天打開這個 VTuber？」

如果答案不夠明確，就先放進提案池，不急著做。
